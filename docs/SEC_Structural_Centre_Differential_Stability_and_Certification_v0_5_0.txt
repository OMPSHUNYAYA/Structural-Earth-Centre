2. STRUCTURAL CENTRE DIFFERENTIAL

Inputs:

left claim structure
right claim structure
left bounded result
right bounded result

The resolver canonicalizes declared dependency maps, compares keys and values, and separately compares the declared profile identifier.

Outputs:

changed_dependencies
changed_dependency_count
result_equivalent
left_claim_id
right_claim_id
left_result_id
right_result_id
differential_state
differential_id

States:

NO_STRUCTURAL_DIFFERENCE
CLAIM_DISTINCT_RESULT_EQUIVALENT
SINGLE_DEPENDENCY_DIVERGENCE
MULTI_DEPENDENCY_DIVERGENCE
UNRESOLVED_COMPARISON

A single changed dependency is eligible for the interpretation:

CONTROLLED_SINGLE_DECLARED_DIFFERENCE

This interpretation is structural and bounded to the supplied comparison contract.

3. CENTRE STABILITY ENVELOPE

Input:

explicit finite samples of exact rational centre coordinates

plus:

a declared perturbation-family description

The sample order is non-authoritative.

Duplicate samples do not alter the resulting region.

Resolution rules:

no admitted samples -> INCOMPLETE

malformed or unsupported rational representation -> UNSUPPORTED

mixed dimensions -> CONFLICT

one distinct point -> RESOLVED_POINT

more than one distinct compatible point -> RESOLVED_REGION

The region is represented as:

AXIS_ALIGNED_EXACT_RATIONAL_ENVELOPE

with:

dimension
lower_bound
upper_bound

The region is the minimal axis-aligned rational box containing the distinct declared sample results.

It does not claim to be a probability region, confidence interval, physical error ellipsoid, or complete uncertainty set.

4. REGION MEMBERSHIP

A point p is a member of the current region R exactly when every axis lies within its inclusive rational bounds.

For dimension k:

lower_i <= p_i <= upper_i for every i in 1..k

Dimension mismatch is not silently coerced.

5. SYMMETRY CERTIFICATE

Inputs:

candidate set
declared equivalence class
optional symmetry breaker

Rules:

empty equivalence class -> INCOMPLETE

breaker outside equivalence class -> CONFLICT

valid breaker -> UNIQUE_CENTRE_ADMITTED

multiple equivalent members with no breaker -> UNIQUE_CENTRE_REFUSED

single-member equivalence class -> UNIQUE_CENTRE_ADMITTED

The certificate identity binds the full canonical certificate body.

Claim identity material contains:

The claim identity intentionally remains distinct from a coordinate or result identity.

7. CENTRE RESOLUTION CERTIFICATE

The certificate can bind an independently established result identity.

Fields include:

claim_id
claim_material
result_id
result_material_hash
result_id_source
resolution_state
resolution_witness
authority_scope
optional symmetry_certificate_id
optional stability_envelope_id
optional evidence_id
certificate_id

result_id_source is:

DECLARED_RESULT_ID

when an existing result identity is supplied, or:

STRUCTURAL_MATERIAL_HASH

when the certificate derives its result reference directly from the supplied result material.

This distinction prevents a certificate from silently replacing a domain-specific result identity with a new unrelated hash.

8. AUTHORITY LIMIT

A certificate is evidence about the bounded declared resolution structure.

It does not:

approve a physical claim outside its authority scope
create a missing dependency
turn an unsupported claim into a supported claim
prove causality from multiple changed dependencies
infer perturbation results that were never admitted

12. DESIGN PRINCIPLE

The innovation layer does not seek novelty by adding terminology alone.

Its executable contribution is the combination of:

controlled dependency differential
bounded region-valued stability
machine-checkable symmetry refusal
claim/result/evidence separation
resolution certificates that preserve existing result identities

within one deterministic structural resolution system.
