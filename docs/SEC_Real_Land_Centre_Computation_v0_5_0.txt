1. PURPOSE

This document defines the two current real-land computations over the included frozen polygon dataset.

The results are bounded profile outputs, not universal claims about the one true geographic centre of Earth.

3. INPUT ADMISSION

Before computation, the resolver validates the input boundary.

Refusal conditions include malformed JSON, invalid text encoding, invalid feature-collection structure, invalid coordinate types, non-finite values, longitude or latitude outside admitted geographic ranges, malformed positions, and invalid rings.

A refused raw input produces:

INPUT_REJECTED

with a deterministic refusal identity.

7. RESOLUTION CERTIFICATES

Each real-land result has a Centre Resolution Certificate.

The certificate preserves the real profile's actual result_id as its declared result identity.

It separately binds the certificate-view result material through result_material_hash.

It also binds the evidence_id.

This prevents the certificate layer from silently replacing the result identity defined by the real-land profile.

10. NUMERICAL LIMIT

The real-land computations use floating-point spherical operations.

They do not claim that every raw intermediate bit is identical across every mathematics library or processor.

The profile authority is instead attached to the 9-decimal identity-quantized structural result.

11. DATASET LIMIT

Changing the dataset bytes changes the dataset identity and creates a new computational boundary.

No current result is automatically transferred to a different dataset.

12. CLAIM LIMIT

Neither real-land profile is a claim to:

physical geocentre
reference-system origin
population centre
gravity centre
universal geographic centre

Each result answers only its declared bounded profile.
